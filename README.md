# 📸 うちの子ギャラリー（Doll Uchinoko Gallery）

このプロジェクトは、ドールやぬいぐるみの「うちの子」を紹介するための、自分専用Webギャラリーを作成するためのテンプレートです。
GitHub PagesとFirebaseを利用することで、**完全に無料**で自分だけのギャラリーを公開・運営できます。

## ✨ このギャラリーでできること
- フィルムロール風の画像表示
- うちの子ごとの詳細プロフィール・カラー設定
- お迎え記念日や誕生日のカウントダウン
- スマホからいつでも写真や文章を投稿・更新

## 🚀 ギャラリーの作り方（かんたん3ステップ）

### 1. このリポジトリをコピー（Fork）する
1. この画面の右上にある **「Fork」** ボタンを押します。
2. そのまま「Create fork」をクリックすると、あなた自身のGitHubアカウントにこのプログラムがコピーされます。

### 2. Firebaseで「データ倉庫」を準備する
※Googleアカウントが必要です。
1. [Firebase Console](https://console.firebase.google.com/) にアクセスし、「プロジェクトを追加」をクリック。
2. プロジェクト名（例: my-gallery）を決めて作成します。
3. 画面中央のウェブアイコン（`</>`）をクリックしてアプリを登録。
4. 表示された `firebaseConfig`（`const firebaseConfig = { ... };` の部分）をコピーしてメモしておきます。
5. 左メニューの「Build」から **Firestore Database** を作成し、ルールを「テストモード」で開始してください。

### 3. 設定を書き換えて公開！
1. 自分のGitHubで `index.html` を開き、編集ボタン（鉛筆アイコン）を押します。
2. コードの最後の方にある `const firebaseConfig = { ... };` の中身を、先ほどメモした自分のものに書き換えます。
3. 「Commit changes...」を押して保存。
4. **Settings > Pages** タブを開き、Build and deployment の Branch が `main` になっていることを確認して保存。
5. 数分待つと、あなた専用のURLが発行されます！

---
### 💡 最初の設定
自分のURLにアクセスすると、最初に「IDとパスワード」を決める画面が出ます。
そこで決めたIDが、あなた専用の管理キーになります。
